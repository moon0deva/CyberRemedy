HOW TO SEND LOGS TO CYBERREMEDY (port 5514)

STEP 1 — Find your CyberRemedy server IP:
  ip addr show | grep "inet " | grep -v 127.0.0.1
  Use that IP in place of SERVER_IP below.

OPTION A — rsyslog (Ubuntu/Debian/CentOS with rsyslog):
  echo "*.* @@SERVER_IP:5514" | sudo tee -a /etc/rsyslog.conf
  sudo systemctl restart rsyslog
  If rsyslog not found: sudo apt install rsyslog -y

OPTION B — syslog-ng (alternative syslog daemon):
  sudo apt install syslog-ng -y
  Add to /etc/syslog-ng/syslog-ng.conf:
  destination d_cyberremedy { tcp("SERVER_IP" port(5514)); };
  log { source(s_src); destination(d_cyberremedy); };
  sudo systemctl restart syslog-ng

OPTION C — logger command (built-in, test immediately):
  logger -n SERVER_IP -P 5514 -T "Test from $(hostname)"
  Send continuously: watch -n 5 'logger -n SERVER_IP -P 5514 -T "heartbeat $(date)"'

OPTION D — Python one-liner (always works, no install):
  python3 -c "import socket,time; s=socket.socket(); s.connect(('SERVER_IP',5514)); [s.send(b'<14>CyberRemedy test '+str(i).encode()+b' ') or time.sleep(2) for i in range(999)]"

OPTION E — netcat stream (pipe any log file):
  tail -f /var/log/syslog | nc SERVER_IP 5514
  tail -f /var/log/auth.log | nc SERVER_IP 5514

WINDOWS — Run the included agent:
  python agent/windows_agent.py --server SERVER_IP --port 5515

QUICK TEST — Verify CyberRemedy is receiving:
  echo "<14>test message from me" | nc -u SERVER_IP 5514
  Then click ↺ Refresh above — the event should appear in the table.
